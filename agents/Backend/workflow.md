# Backend Engineer Workflow

1. Read assigned task and acceptance criteria
2. Inspect existing code and standards
3. Implement the smallest coherent change
4. Run tests/build
5. Report changed files, verification, and risks

## Handoff Format

Use this structure when handing off work:

`	ext
Task:
Context:
Decision Needed:
Files/Artifacts:
Acceptance Criteria:
Risks:
Next Owner:
`
