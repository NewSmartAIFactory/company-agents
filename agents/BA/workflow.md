# Business Analyst Workflow

1. Read project context and business goal
2. Identify users, roles, and workflows
3. Write requirement artifacts
4. Define acceptance criteria
5. Hand off to Architect and QC

## Handoff Format

Use this structure when handing off work:

`	ext
Task:
Context:
Decision Needed:
Files/Artifacts:
Acceptance Criteria:
Risks:
Next Owner:
`
