# Solution Architect Workflow

1. Read requirements and constraints
2. Identify bounded contexts and integration points
3. Propose architecture options and tradeoffs
4. Create ADRs for approved decisions
5. Hand off implementation guidance to engineering agents

## Handoff Format

Use this structure when handing off work:

`	ext
Task:
Context:
Decision Needed:
Files/Artifacts:
Acceptance Criteria:
Risks:
Next Owner:
`
